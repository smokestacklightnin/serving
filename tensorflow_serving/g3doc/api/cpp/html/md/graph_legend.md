Graph Legend
:::
:::
:::

::: {.contents}
This page explains how to interpret the graphs that are generated by
doxygen.

Consider the following example:

::: {.fragment}
::: {.line}
[/\*! Invisible class because of truncation \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}Invisible { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Truncated class, inheritance relation is hidden \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}Truncated : [public]{.keyword} Invisible { };
:::

::: {.line}
:::

::: {.line}
[/\* Class not documented with doxygen comments \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}Undocumented { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Class that is inherited using public inheritance \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}PublicBase : [public]{.keyword} Truncated { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! A template class \*/]{.comment}
:::

::: {.line}
[template]{.keyword}\<[class]{.keyword} T\> [class ]{.keyword}Templ { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Class that is inherited using protected inheritance \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}ProtectedBase { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Class that is inherited using private inheritance \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}PrivateBase { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Class that is used by the Inherited class \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}Used { };
:::

::: {.line}
[]{.comment}
:::

::: {.line}
[/\*! Super class that inherits a number of other classes \*/]{.comment}
:::

::: {.line}
[class ]{.keyword}Inherited : [public]{.keyword} PublicBase,
:::

::: {.line}
[protected]{.keyword} ProtectedBase,
:::

::: {.line}
[private]{.keyword} PrivateBase,
:::

::: {.line}
[public]{.keyword} Undocumented,
:::

::: {.line}
[public]{.keyword} Templ\<int\>
:::

::: {.line}
{
:::

::: {.line}
[private]{.keyword}:
:::

::: {.line}
Used \*m\_usedClass;
:::

::: {.line}
};
:::
:::

This will result in the following graph:

![](graph_legend.png){.inline}

The boxes in the above graph have the following meaning:

-   A filled gray box represents the struct or class for which the graph
    is generated.
-   A box with a black border denotes a documented struct or class.
-   A box with a gray border denotes an undocumented struct or class.
-   A box with a red border denotes a documented struct or class
    forwhich not all inheritance/containment relations are shown. A
    graph is truncated if it does not fit within the specified
    boundaries.

The arrows have the following meaning:

-   A dark blue arrow is used to visualize a public inheritance relation
    between two classes.
-   A dark green arrow is used for protected inheritance.
-   A dark red arrow is used for private inheritance.
-   A purple dashed arrow is used if a class is contained or used by
    another class. The arrow is labelled with the variable(s) through
    which the pointed class or struct is accessible.
-   A yellow dashed arrow denotes a relation between a template instance
    and the template class it was instantiated from. The arrow is
    labelled with the template parameters of the instance.
