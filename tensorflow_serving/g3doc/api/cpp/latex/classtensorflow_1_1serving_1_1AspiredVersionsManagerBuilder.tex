\doxysubsection{Detailed Description}
Builds an \mbox{\hyperlink{classtensorflow_1_1serving_1_1AspiredVersionsManager}{Aspired\+Versions\+Manager}} with options and sources connected to it. It takes over the ownership of the sources and the returned manager handles the destruction of itself and its dependencies. Both single sources and source/source-\/adapter chains are accepted, i.\+e. you can use sources that directly supply loaders (\mbox{\hyperlink{classtensorflow_1_1serving_1_1Source}{Source}}\&lt;std\+::unique\+\_\+ptr\&lt;\mbox{\hyperlink{classtensorflow_1_1serving_1_1Loader}{Loader}}$>$$>$) or composites that consist of \mbox{\hyperlink{classtensorflow_1_1serving_1_1Source}{Source}}\&lt;S$>$ + some chain of \mbox{\hyperlink{classtensorflow_1_1serving_1_1SourceAdapter}{Source\+Adapter}}\&lt;S, ...$>$, ..., \mbox{\hyperlink{classtensorflow_1_1serving_1_1SourceAdapter}{Source\+Adapter}}\&lt;..., std\+::unique\+\_\+ptr\&lt;\mbox{\hyperlink{classtensorflow_1_1serving_1_1Loader}{Loader}}$>$$>$. The builder connects the chain for you.

Usage\+: \begin{DoxyVerb}...
AspiredVersionsManagerBuilder::Options options = ManagerOptions();
std::unique_ptr&lt;AspiredVersionsManagerBuilder> builder;
TF_CHECK_OK(AspiredVersionsManagerBuilder::Create(
    std::move(options), &builder));
builder->AddSource(std::move(some_source));
builder->AddSourceChain(
    std::move(source), std::move(source_adapter1),
    std::move(source_adapter2));
std::unique_ptr&lt;Manager> manager = builder->Build();
...
\end{DoxyVerb}
 NOTE\+: A builder can only be used to build a single \mbox{\hyperlink{classtensorflow_1_1serving_1_1AspiredVersionsManager}{Aspired\+Versions\+Manager}}.

This class is not thread-\/safe.
