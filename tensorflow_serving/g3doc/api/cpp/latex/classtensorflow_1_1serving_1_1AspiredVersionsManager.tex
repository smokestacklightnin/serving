\doxysubsection{Detailed Description}
A manager that implements the \mbox{\hyperlink{classtensorflow_1_1serving_1_1Target}{Target}}\&lt;\mbox{\hyperlink{classtensorflow_1_1serving_1_1Loader}{Loader}}$>$ API which uses aspired-\/versions callbacks to dictate which servable versions to load. This manager also uses that API to infer which ones to unload\+: If a given servable version is currently loaded, and is omitted from an aspired-\/versions callback invocation pertaining to its servable stream, this manager interprets that omission as an implicit instruction to unload the version. See below for details.

(The implicit-\/unload semantics facilitates stateless \mbox{\hyperlink{classtensorflow_1_1serving_1_1Source}{Source}} implementations, whereby a given iteration of the \mbox{\hyperlink{classtensorflow_1_1serving_1_1Source}{Source}}\textquotesingle{}s logic simply decides which versions of a servable ought to be loaded, without needing to know what it has decided in the past.)

This manager makes transitions between versions of a servable stream using a configured \mbox{\hyperlink{classtensorflow_1_1serving_1_1AspiredVersionPolicy}{Aspired\+Version\+Policy}}. The manager prefers unloading before loading to free up resources in the server when deciding among transitions suggested by the policy.
