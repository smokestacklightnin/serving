\doxysubsection{Detailed Description}
A factory that creates Saved\+Model\+Bundles from Saved\+Model or \mbox{\hyperlink{structtensorflow_1_1serving_1_1SessionBundle}{Session\+Bundle}} export paths.

The emitted sessions only support Run(), and although not enforced it is expected that the client will only make non-\/mutating Run() calls. (If this restriction, which we\textquotesingle{}ve added as a safety measure, is problematic for your use-\/case please contact the Tensor\+Flow Serving team to discuss disabling it.)

If the config calls for batching, the emitted sessions automatically batch Run() calls behind the scenes, using a Shared\+Batch\+Scheduler owned by the factory. The \textquotesingle{}config.\+num\+\_\+batch\+\_\+threads\textquotesingle{} threads are shared across all session instances created by this factory. However, each session has its own dedicated queue of size \textquotesingle{}config.\+max\+\_\+enqueued\+\_\+batches\textquotesingle{}.

The factory can also estimate the resource (e.\+g. RAM) requirements of a Saved\+Model\+Bundle based on the Saved\+Model (i.\+e. prior to loading the session).

This class is thread-\/safe.
